Laravel Payment Gateway

A reusable, multi-gateway Laravel payment package supporting Iranian banking system. Designed for flexibility and ease of use.

Requirements: PHP ^8.2, Laravel ^11.0 | ^12.0

Installation

Install the package via composer:

composer require efati/payment-gateway

Then run the installation command to scaffold the package: 

php artisan payment:install

This command will publish config, migrations, seeders, models, and core logic to your application.

Finally, run migrations and seeders:

php artisan migrate
php artisan db:seed --class=PaymentGateway\Database\Seeders\BankSeeder
php artisan db:seed --class=PaymentGateway\Database\Seeders\GatewaySeeder

Configuration

The configuration file is located at config/payment.php. You can set your gateway credentials in your .env file: 

ZARINPAL_MERCHANT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MELLAT_TERMINAL_ID=xxxxxxx
MELLAT_USERNAME=xxxxxxx
MELLAT_PASSWORD=xxxxxxx

Usage

Initialization

use PaymentGateway\Facades\Payment;
use App\Payment\DTOs\PaymentRequestDTO;

$dto = new PaymentRequestDTO(
    amount: 10000,
    orderId: 'ORD-123',
    callbackUrl: route('payment.callback'),
    description: 'Order payment',
    mobile: '09123456789'
);

// Use Default Gateway
$result = Payment::initialize($dto);

// Use Specific Gateway
$result = Payment::zarinpal()->initialize($dto);

return redirect($result['url']);

Verification

use PaymentGateway\Facades\Payment;
use App\Payment\DTOs\PaymentVerifyDTO;

$dto = new PaymentVerifyDTO(
    amount: 10000,
    authority: request('Authority'),
    gateway: 'zarinpal'
);

try {
    $receipt = Payment::verify($dto);
    return "Success! Ref ID: " . $receipt['ref_id'];
} catch (\Exception $e) {
    return "Failed: " . $e->getMessage();
}

Transaction Persistence

The package automatically handles transaction logging in the database.

  • Initialize: Creates a transaction with status pending.
  • Verify: Updates the transaction status to success or failed and records the reference ID.

You can access transactions via the PaymentTransaction model (published to app/Models).

Customization

The core logic is published to app/Payment/. You can modify any file in this directory to change the behavior of the package.

  • app/Payment/PaymentManager.php: Main entry point.
  • app/Payment/Gateways/: Gateway implementations.
  • app/Payment/DTOs/: Data Transfer Objects.
