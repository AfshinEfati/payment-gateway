درگاه پرداخت لاراول

یک پکیج قابل استفاده مجدد و چند درگاهی برای لاراول که از سیستم بانکی ایران پشتیبانی می‌کند. طراحی شده برای انعطاف‌پذیری و سهولت استفاده.

نیازمندی‌ها: PHP ^8.2, Laravel ^11.0 | ^12.0

نصب و راه اندازی

پکیج را از طریق کامپوزر نصب کنید:

composer require efati/payment-gateway

سپس دستور نصب را اجرا کنید تا پکیج اسکفولد شود:

php artisan payment:install

این دستور کانفیگ، مایگریشن‌ها، سیدرها، مدل‌ها و منطق اصلی را در برنامه شما منتشر می‌کند.

در نهایت، مایگریشن‌ها و سیدرها را اجرا کنید:

php artisan migrate
php artisan db:seed --class=PaymentGateway\Database\Seeders\BankSeeder
php artisan db:seed --class=PaymentGateway\Database\Seeders\GatewaySeeder

پیکربندی

فایل پیکربندی در مسیر config/payment.php قرار دارد. می‌توانید اطلاعات درگاه خود را در فایل .env تنظیم کنید:

ZARINPAL_MERCHANT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MELLAT_TERMINAL_ID=xxxxxxx
MELLAT_USERNAME=xxxxxxx
MELLAT_PASSWORD=xxxxxxx

نحوه استفاده

ایجاد درخواست پرداخت

use PaymentGateway\Facades\Payment;
use App\Payment\DTOs\PaymentRequestDTO;

$dto = new PaymentRequestDTO(
    amount: 10000,
    orderId: 'ORD-123',
    callbackUrl: route('payment.callback'),
    description: 'Order payment',
    mobile: '09123456789'
);

// استفاده از درگاه پیش‌فرض
$result = Payment::initialize($dto);

// استفاده از درگاه خاص
$result = Payment::zarinpal()->initialize($dto);

return redirect($result['url']);

تایید پرداخت

use PaymentGateway\Facades\Payment;
use App\Payment\DTOs\PaymentVerifyDTO;

$dto = new PaymentVerifyDTO(
    amount: 10000,
    authority: request('Authority'),
    gateway: 'zarinpal'
);

try {
    $receipt = Payment::verify($dto);
    return "موفق! کد رهگیری: " . $receipt['ref_id'];
} catch (\Exception $e) {
    return "ناموفق: " . $e->getMessage();
}

ذخیره تراکنش‌ها

پکیج به صورت خودکار لاگ تراکنش‌ها را در دیتابیس مدیریت می‌کند.

  • ایجاد: یک تراکنش با وضعیت pending ایجاد می‌کند.
  • تایید: وضعیت تراکنش را به success یا failed بروزرسانی کرده و کد رهگیری را ثبت می‌کند.

می‌توانید از طریق مدل PaymentTransaction (منتشر شده در app/Models) به تراکنش‌ها دسترسی داشته باشید.

شخصی سازی

منطق اصلی در مسیر app/Payment/ منتشر می‌شود. می‌توانید هر فایلی در این دایرکتوری را برای تغییر رفتار پکیج ویرایش کنید.

  • app/Payment/PaymentManager.php: نقطه ورود اصلی.
  • app/Payment/Gateways/: پیاده‌سازی درگاه‌ها.
  • app/Payment/DTOs/: اشیاء انتقال داده.
